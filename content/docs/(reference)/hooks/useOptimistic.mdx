## Справка {/*reference*/}

### `useOptimistic(value, reducer?)` {/*useoptimistic*/}

Вызовите `useOptimistic` на верхнем уровне компонента, чтобы создать для значения оптимистичное состояние.
```js
import { useOptimistic } from 'react';

function MyComponent({name, todos}) {
  const [optimisticAge, setOptimisticAge] = useOptimistic(28);
  const [optimisticName, setOptimisticName] = useOptimistic(name);
  const [optimisticTodos, setOptimisticTodos] = useOptimistic(todos, todoReducer);
  // ...
}
```

#### Параметры {/*parameters*/}

* `value`: Значение, которое возвращается, когда нет ожидающих Actions.
* **optional** `reducer(currentState, action)`: Функция reducer, которая определяет, как обновляется оптимистичное состояние. Она должна быть чистой, принимать текущее состояние и аргументы action reducer-а и возвращать следующее оптимистичное состояние.

#### Возвращает {/*returns*/}

`useOptimistic` возвращает массив ровно из двух значений:

1. `optimisticState`: Текущее оптимистичное состояние. Оно равно `value`, если Action не ожидает выполнения; иначе оно равно состоянию, возвращённому `reducer` (или значению, переданному в set-функцию, если `reducer` не был указан).
2. [`set`-функция](#setoptimistic), которая позволяет обновлять оптимистичное состояние до другого значения внутри Action.

---

### `set`-функции, например `setOptimistic(optimisticState)` {/*setoptimistic*/}

`set`-функция, возвращаемая `useOptimistic`, позволяет обновлять состояние на время [Action](reference/react/useTransition#functions-called-in-starttransition-are-called-actions). Вы можете передать следующее состояние напрямую или функцию, которая вычисляет его из предыдущего состояния:

```js
const [optimisticLike, setOptimisticLike] = useOptimistic(false);
const [optimisticSubs, setOptimisticSubs] = useOptimistic(subs);

function handleClick() {
  startTransition(async () => {
    setOptimisticLike(true);
    setOptimisticSubs(a => a + 1);
    await saveChanges();
  });
}
```

#### Параметры {/*setoptimistic-parameters*/}

* `optimisticState`: Значение, которое вы хотите видеть как оптимистичное состояние во время [Action](reference/react/useTransition#functions-called-in-starttransition-are-called-actions). Если вы передали в `useOptimistic` `reducer`, это значение будет передано ему вторым аргументом. Оно может быть любого типа.
    * Если вы передадите в `optimisticState` функцию, она будет считаться _updater function_. Она должна быть чистой, принимать pending state как единственный аргумент и возвращать следующее оптимистичное состояние. React поместит вашу updater function в очередь и выполнит повторный рендер компонента. Во время следующего рендера React вычислит следующее состояние, применяя очередь updater-ов к предыдущему состоянию — так же, как [updaters в `useState`](/reference/react/useState#setstate-parameters).

#### Возвращает {/*setoptimistic-returns*/}

`set`-функции ничего не возвращают.

#### Предостережения {/*setoptimistic-caveats*/}

* `set`-функцию нужно вызывать внутри [Action](reference/react/useTransition#functions-called-in-starttransition-are-called-actions). Если вызвать setter вне Action, [React покажет предупреждение](#an-optimistic-state-update-occurred-outside-a-transition-or-action), и оптимистичное состояние ненадолго отрендерится.

<Callout>

#### Как работает оптимистичное состояние {/*how-optimistic-state-works*/}

`useOptimistic` позволяет показать временное значение, пока Action выполняется:

```js
const [value, setValue] = useState('a');
const [optimistic, setOptimistic] = useOptimistic(value);

startTransition(async () => {
  setOptimistic('b');
  const newValue = await saveChanges('b');
  setValue(newValue);
});
```

Когда setter вызывается внутри Action, `useOptimistic` вызывает повторный рендер, чтобы показать это состояние, пока Action выполняется. В противном случае возвращается `value`, переданный в `useOptimistic`.

Это состояние называется «оптимистичным», потому что оно позволяет сразу показать пользователю результат выполнения Action, хотя само действие занимает время.

**Как проходит обновление**

1. **Мгновенное обновление**: Когда вызывается `setOptimistic('b')`, React сразу рендерит `'b'`.

2. **(Необязательно) ожидание внутри Action**: Если вы используете `await` внутри Action, React продолжает показывать `'b'`.

3. **Планируется Transition**: `setValue(newValue)` планирует обновление реального состояния.

4. **(Необязательно) ожидание Suspense**: Если `newValue` уходит в suspend, React продолжает показывать `'b'`.

5. **Единственный коммит рендера**: В конце `newValue` коммитится и для `value`, и для `optimistic`.

Дополнительного рендера, чтобы «очистить» оптимистичное состояние, не нужно. Оптимистичное и реальное состояние сходятся в одном и том же рендере, когда Transition завершается.

<Callout type='idea'>

#### Оптимистичное состояние временное {/*optimistic-state-is-temporary*/}

Оптимистичное состояние отображается только пока Action выполняется; в остальное время отображается `value`.

Если `saveChanges` вернёт `'c'`, то и `value`, и `optimistic` будут равны `'c'`, а не `'b'`.

</Callout>

**Как определяется итоговое состояние**

Аргумент `value` у `useOptimistic` определяет, что показывать после завершения Action. Это зависит от выбранного вами паттерна:

- **Жёстко заданные значения**, например `useOptimistic(false)`: после Action `state` по-прежнему `false`, поэтому UI покажет `false`. Это удобно для pending-состояний, где вы всегда стартуете с `false`.

- **Переданные props или state**, например `useOptimistic(isLiked)`: если родитель обновит `isLiked` во время Action, после завершения Action будет использовано новое значение. Так UI отражает результат Action.

- **Паттерн reducer**, например `useOptimistic(items, fn)`: если `items` изменятся, пока Action ожидает завершения, React снова запустит ваш `reducer` с новыми `items`, чтобы пересчитать состояние. Это сохраняет ваши оптимистичные добавления поверх самых свежих данных.

**Что происходит, если Action завершается с ошибкой**

Если Action выбросит ошибку, Transition всё равно завершится, и React отрендерит текущее `value`. Обычно родитель обновляет `value` только при успехе, поэтому при ошибке `value` не меняется, и UI показывает то же, что и до оптимистичного обновления. Ошибку можно поймать и показать сообщение пользователю.

</Callout>

## Использование {/*usage*/}

### Добавление оптимистичного состояния в компонент {/*adding-optimistic-state-to-a-component*/}

Вызовите `useOptimistic` на верхнем уровне компонента, чтобы объявить одно или несколько оптимистичных состояний.

```js 
import { useOptimistic } from 'react';

function MyComponent({age, name, todos}) {
  const [optimisticAge, setOptimisticAge] = useOptimistic(age);
  const [optimisticName, setOptimisticName] = useOptimistic(name);
  const [optimisticTodos, setOptimisticTodos] = useOptimistic(todos, reducer);
  // ...
```

`useOptimistic` возвращает массив ровно из двух элементов:

1. Оптимистичное состояние, изначально равное переданному value.
2. set-функция, которая позволяет временно менять состояние во время [Action](reference/react/useTransition#functions-called-in-starttransition-are-called-actions).
   * Если указан reducer, он будет вызван перед возвратом оптимистичного состояния.

Чтобы использовать оптимистичное состояние, вызывайте `set`-функцию внутри Action.

Actions — это функции, вызываемые внутри `startTransition`:

```js
function onAgeChange(e) {
  startTransition(async () => {
    setOptimisticAge(42);
    const newAge = await postAge(42);
    setAge(newAge);
  });
}
```

React сначала отрендерит оптимистичное состояние `42`, пока `age` остаётся текущим значением. Action ждёт POST, а затем рендерит `newAge` и для `age`, и для `optimisticAge`.

См. [Как работает оптимистичное состояние](#how-optimistic-state-works), если хотите разобрать это глубже.

<Callout>

При использовании [Action props](/reference/react/useTransition#exposing-action-props-from-components) set-функцию можно вызывать без `startTransition`:

```js 
async function submitAction() {
  setOptimisticName('Taylor');
  await updateName('Taylor');
}
```

Это работает потому, что Action props и так вызываются внутри `startTransition`.

Пример см.: [Использование оптимистичного состояния в Action props](#using-optimistic-state-in-action-props).

</Callout>

### Использование оптимистичного состояния в Action props {/*using-optimistic-state-in-action-props*/}

В [Action prop](/reference/react/useTransition#exposing-action-props-from-components) можно напрямую вызывать optimistic setter без `startTransition`.

В этом примере оптимистичное состояние устанавливается внутри `submitAction` prop у `<form>`:

```js tab="Tab 1"
//App.js
import { useState, startTransition } from 'react';
import EditName from './EditName';

export default function App() {
  const [name, setName] = useState('Alice');

  return <EditName name={name} action={setName} />;
}
```
```js tab="Tab 2"
//EditName.js active
import { useOptimistic, startTransition } from 'react';
import { updateName } from './actions.js';

export default function EditName({ name, action }) {
  const [optimisticName, setOptimisticName] = useOptimistic(name);

  async function submitAction(formData) {
    const newName = formData.get('name');
    setOptimisticName(newName);

    const updatedName = await updateName(newName);
    startTransition(() => {
      action(updatedName);
    })
  }

  return (
    <form action={submitAction}>
      <p>Your name is: {optimisticName}</p>
      <p>
        <label>Change it: </label>
        <input
          type="text"
          name="name"
          disabled={name !== optimisticName}
        />
      </p>
    </form>
  );
}
```
```js tab="Tab 3"
//actions.js hidden
export async function updateName(name) {
  await new Promise((res) => setTimeout(res, 1000));
  return name;
}
```

В этом примере, когда пользователь отправляет форму, `optimisticName` сразу обновляется и показывает `newName` оптимистично, пока запрос к серверу ещё выполняется. Когда запрос завершается, `name` и `optimisticName` рендерятся с фактическим `updatedName` из ответа.

<Callout>

#### Почему здесь не нужен `startTransition`? {/*why-doesnt-this-need-starttransition*/}

По соглашению props, вызываемые внутри `startTransition`, называют словами с «Action».

Поскольку `submitAction` назван с «Action», вы уже знаете, что он вызывается внутри `startTransition`.

См. [Exposing `action` prop from components](/reference/react/useTransition#exposing-action-props-from-components) для паттерна Action prop.

</Callout>

### Добавление оптимистичного состояния в Action props {/*adding-optimistic-state-to-action-props*/}

Когда вы создаёте [Action prop](/reference/react/useTransition#exposing-action-props-from-components), можно добавить `useOptimistic`, чтобы сразу показать обратную связь.

Вот кнопка, которая показывает «Submitting…», пока `action` ожидается:

```js tab="Tab 1"
//App.js
import { useState, startTransition } from 'react';
import Button from './Button';
import { submitForm } from './actions.js';

export default function App() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <Button action={async () => {
        await submitForm();
        startTransition(() => {
          setCount(c => c + 1);
        });
      }}>Increment</Button>
      {count > 0 && <p>Submitted {count}!</p>}
    </div>
  );
}
```
```js tab="Tab 2"
//Button.js active
import { useOptimistic, startTransition } from 'react';

export default function Button({ action, children }) {
  const [isPending, setIsPending] = useOptimistic(false);

  return (
    <button
      disabled={isPending}
      onClick={() => {
        startTransition(async () => {
          setIsPending(true);
          await action();
        });
      }}
    >
      {isPending ? 'Submitting...' : children}
    </button>
  );
}
```
```js tab="Tab 3"
//actions.js hidden
export async function submitForm() {
  await new Promise((res) => setTimeout(res, 1000));
}
```

Когда кнопку нажимают, `setIsPending(true)` использует оптимистичное состояние, чтобы сразу показать «Submitting…» и отключить кнопку. Когда Action завершится, `isPending` автоматически станет `false`.

Этот паттерн автоматически показывает pending-состояние, как бы ни использовался `action` prop у `Button`:

```js
// Show pending state for a state update
<Button action={() => { setState(c => c + 1) }} />

// Show pending state for a navigation
<Button action={() => { navigate('/done') }} />

// Show pending state for a POST
<Button action={async () => { await fetch(/* ... */) }} />

// Show pending state for any combination
<Button action={async () => {
  setState(c => c + 1);
  await fetch(/* ... */);
  navigate('/done');
}} />
```

Состояние ожидания будет показываться, пока не завершится всё, что происходит внутри `action` prop.

<Callout>

Также можно использовать [`useTransition`](/reference/react/useTransition), чтобы получить pending-состояние через `isPending`.

Разница в том, что `useTransition` даёт вам функцию `startTransition`, а `useOptimistic` работает с любым Transition. Используйте то, что лучше подходит вашему компоненту.

</Callout>

### Оптимистичное обновление props или state {/*updating-props-or-state-optimistically*/}

Вы можете обернуть props или state в `useOptimistic`, чтобы обновлять их сразу, пока Action выполняется.

```js tab="Tab 1"
//App.js
import { useState, useOptimistic, startTransition } from 'react';
import { toggleLike } from './actions.js';

export default function App() {
  const [isLiked, setIsLiked] = useState(false);
  const [optimisticIsLiked, setOptimisticIsLiked] = useOptimistic(isLiked);

  function handleClick() {
    startTransition(async () => {
      const newValue = !optimisticIsLiked
      console.log('⏳ setting optimistic state: ' + newValue);

      setOptimisticIsLiked(newValue);
      const updatedValue = await toggleLike(newValue);

      startTransition(() => {
        console.log('⏳ setting real state: ' + updatedValue );
        setIsLiked(updatedValue);
      });
    });
  }

  if (optimisticIsLiked !== isLiked) {
    console.log('✅ rendering optimistic state: ' + optimisticIsLiked);
  } else {
    console.log('✅ rendering real value: ' + optimisticIsLiked);
  }


  return (
    <button onClick={handleClick}>
      {optimisticIsLiked ? '❤️ Unlike' : '🤍 Like'}
    </button>
  );
}
```
```js tab="Tab 2"
//actions.js hidden
export async function toggleLike(value) {
  return await new Promise((res) => setTimeout(() => res(value), 1000));
  // In a real app, this would update the server
}
```
```js tab="Tab 3"
//index.js hidden
import React from 'react';
import {createRoot} from 'react-dom/client';
import './styles.css';

import App from './App';

const root = createRoot(document.getElementById('root'));
// Not using StrictMode so double render logs are not shown.
root.render(<App />);
```

Когда кнопку нажимают, `setOptimisticIsLiked` сразу обновляет отображаемое состояние, показывая сердечко как отмеченное. Тем временем `await toggleLike` выполняется в фоне. Когда `await` завершается, `setIsLiked` в родителе обновляет «реальное» состояние `isLiked`, а оптимистичное состояние рендерится так, чтобы совпасть с новым значением.

<Callout>

В этом примере для вычисления следующего значения используется `optimisticIsLiked`. Это работает, когда базовое состояние не меняется, но если базовое состояние может измениться, пока Action ожидается, лучше использовать updater функции или reducer.

См. [Обновление state на основе текущего состояния](#updating-state-based-on-current-state) для примера.

</Callout>

### Обновление нескольких значений вместе {/*updating-multiple-values-together*/}

Когда оптимистичное обновление затрагивает несколько связанных значений, используйте reducer, чтобы обновлять их вместе. Так UI остаётся согласованным.

Вот кнопка подписки, которая обновляет и состояние подписки, и счётчик подписчиков:

```js tab="Tab 1"
//App.js
import { useState, startTransition } from 'react';
import { followUser, unfollowUser } from './actions.js';
import FollowButton from './FollowButton';

export default function App() {
  const [user, setUser] = useState({
    name: 'React',
    isFollowing: false,
    followerCount: 10500
  });

  async function followAction(shouldFollow) {
    if (shouldFollow) {
      await followUser(user.name);
    } else {
      await unfollowUser(user.name);
    }
    startTransition(() => {
      setUser(current => ({
        ...current,
        isFollowing: shouldFollow,
        followerCount: current.followerCount + (shouldFollow ? 1 : -1)
      }));
    });
  }

  return <FollowButton user={user} followAction={followAction} />;
}
```
```js tab="Tab 2"
//FollowButton.js active
import { useOptimistic, startTransition } from 'react';

export default function FollowButton({ user, followAction }) {
  const [optimisticState, updateOptimistic] = useOptimistic(
    { isFollowing: user.isFollowing, followerCount: user.followerCount },
    (current, isFollowing) => ({
      isFollowing,
      followerCount: current.followerCount + (isFollowing ? 1 : -1)
    })
  );

  function handleClick() {
    const newFollowState = !optimisticState.isFollowing;
    startTransition(async () => {
      updateOptimistic(newFollowState);
      await followAction(newFollowState);
    });
  }

  return (
    <div>
      <p><strong>{user.name}</strong></p>
      <p>{optimisticState.followerCount} followers</p>
      <button onClick={handleClick}>
        {optimisticState.isFollowing ? 'Unfollow' : 'Follow'}
      </button>
    </div>
  );
}
```
```js tab="Tab 3"
//actions.js hidden
export async function followUser(name) {
  await new Promise((res) => setTimeout(res, 1000));
}

export async function unfollowUser(name) {
  await new Promise((res) => setTimeout(res, 1000));
}
```


Reducer получает новое значение `isFollowing` и вычисляет и новое состояние подписки, и обновлённый счётчик подписчиков за один update. Это гарантирует, что текст кнопки и счётчик всегда будут синхронизированы.


#### Выбор между updater-ами и reducer-ами {/*choosing-between-updaters-and-reducers*/}

`useOptimistic` поддерживает два паттерна вычисления состояния на основе текущего состояния:

**Updater-функции** работают так же, как [updaters в `useState`](/reference/react/useState#updating-state-based-on-the-previous-state). Передайте функцию в setter:

```js
const [optimistic, setOptimistic] = useOptimistic(value);
setOptimistic(current => !current);
```

**Reducers** отделяют логику обновления от вызова setter-а:

```js
const [optimistic, dispatch] = useOptimistic(value, (current, action) => {
  // Calculate next state based on current and action
});
dispatch(action);
```

**Используйте updater-ы** там, где вызов setter-а естественным образом описывает обновление. Это похоже на `setState(prev => ...)` в `useState`.

**Используйте reducer-ы** там, где нужно передавать данные для обновления (например, какой item добавить) или когда одним hook-ом нужно обрабатывать несколько типов обновлений.

**Зачем использовать reducer?**

Reducer особенно важен, когда базовое состояние может измениться, пока Transition ожидается. Если `todos` изменятся, пока добавление ещё pending (например, другой пользователь добавил todo), React снова запустит ваш reducer с новыми `todos`, чтобы пересчитать, что показывать. Так новый todo добавится в самый свежий список, а не в устаревшую копию.

Updater-функция вроде `setOptimistic(prev => [...prev, newItem])` увидит только состояние на момент начала Transition и не заметит изменения, которые произошли во время async-работы.

```js tab="Tab 1"
//App.js
import { useState, startTransition } from 'react';
import { addTodo } from './actions.js';
import TodoList from './TodoList';

export default function App() {
  const [todos, setTodos] = useState([
    { id: 1, text: 'Learn React' }
  ]);

  async function addTodoAction(newTodo) {
    const savedTodo = await addTodo(newTodo);
    startTransition(() => {
      setTodos(todos => [...todos, savedTodo]);
    });
  }

  return <TodoList todos={todos} addTodoAction={addTodoAction} />;
}
```

```js tab="Tab 2"
  const [optimisticTodos, addOptimisticTodo] = useOptimistic(

    (currentTodos, newTodo) => [
      ...currentTodos,
      { id: newTodo.id, text: newTodo.text, pending: true }
    ]

  function handleAddTodo(text) {
    const newTodo = { id: crypto.randomUUID(), text: text };
    startTransition(async () => {
      addOptimisticTodo(newTodo);
      await addTodoAction(newTodo);
    });
  }

  return (
    <div>
      <button onClick={() => handleAddTodo('New todo')}>Add Todo</button>
      <ul>
        {optimisticTodos.map(todo => (
          <li key={todo.id}>
            {todo.text} {todo.pending && "(Adding...)"}
          </li>
        ))}
      </ul>
    </div>
  );
}
```

```js tab="Tab 3"

  // In a real app, this would save to the server
  return { ...todo, pending: false };
}
```

Reducer получает текущий список todos и новый todo, который нужно добавить. Это важно, потому что если `todos` изменятся, пока добавление pending (например, другой пользователь добавил todo), React обновит оптимистичное состояние, снова запустив reducer на обновлённом списке. Так новый todo попадёт в самый свежий список, а не в устаревшую копию.

<Callout>

Каждый оптимистичный item включает флаг `pending: true`, чтобы можно было показывать состояние загрузки для отдельных элементов. Когда сервер отвечает, а родитель обновляет канонический список `todos` сохранённым элементом, оптимистичное состояние обновляется до подтверждённого элемента без флага pending.

</Callout>

### Обработка нескольких типов `action` {/*handling-multiple-action-types*/}

Когда нужно обрабатывать несколько типов оптимистичных обновлений (например, добавление и удаление), используйте reducer-паттерн с объектами `action`.

В этом примере корзины покупок показано, как обрабатывать добавление и удаление одним reducer-ом:

```js tab="Tab 1"
//App.js
import { useState, startTransition } from 'react';
import { addToCart, removeFromCart, updateQuantity } from './actions.js';
import ShoppingCart from './ShoppingCart';

export default function App() {
  const [cart, setCart] = useState([]);

  const cartActions = {
    async add(item) {
      await addToCart(item);
      startTransition(() => {
        setCart(current => {
          const exists = current.find(i => i.id === item.id);
          if (exists) {
            return current.map(i =>
              i.id === item.id ? { ...i, quantity: i.quantity + 1 } : i
            );
          }
          return [...current, { ...item, quantity: 1 }];
        });
      });
    },
    async remove(id) {
      await removeFromCart(id);
      startTransition(() => {
        setCart(current => current.filter(item => item.id !== id));
      });
    },
    async updateQuantity(id, quantity) {
      await updateQuantity(id, quantity);
      startTransition(() => {
        setCart(current =>
          current.map(item =>
            item.id === id ? { ...item, quantity } : item
          )
        );
      });
    }
  };

  return <ShoppingCart cart={cart} cartActions={cartActions} />;
}
```
```js tab="Tab 2"
//ShoppingCart.js active
import { useOptimistic, startTransition } from 'react';

export default function ShoppingCart({ cart, cartActions }) {
  const [optimisticCart, dispatch] = useOptimistic(
    cart,
    (currentCart, action) => {
      switch (action.type) {
        case 'add':
          const exists = currentCart.find(item => item.id === action.item.id);
          if (exists) {
            return currentCart.map(item =>
              item.id === action.item.id
                ? { ...item, quantity: item.quantity + 1, pending: true }
                : item
            );
          }
          return [...currentCart, { ...action.item, quantity: 1, pending: true }];
        case 'remove':
          return currentCart.filter(item => item.id !== action.id);
        case 'update_quantity':
          return currentCart.map(item =>
            item.id === action.id
              ? { ...item, quantity: action.quantity, pending: true }
              : item
          );
        default:
          return currentCart;
      }
    }
  );

  function handleAdd(item) {
    startTransition(async () => {
      dispatch({ type: 'add', item });
      await cartActions.add(item);
    });
  }

  function handleRemove(id) {
    startTransition(async () => {
      dispatch({ type: 'remove', id });
      await cartActions.remove(id);
    });
  }

  function handleUpdateQuantity(id, quantity) {
    startTransition(async () => {
      dispatch({ type: 'update_quantity', id, quantity });
      await cartActions.updateQuantity(id, quantity);
    });
  }

  const total = optimisticCart.reduce(
    (sum, item) => sum + item.price * item.quantity,
    0
  );

  return (
    <div>
      <h2>Shopping Cart</h2>
      <div style={{ marginBottom: 16 }}>
        <button onClick={() => handleAdd({
          id: 1, name: 'T-Shirt', price: 25
        })}>
          Add T-Shirt ($25)
        </button>{' '}
        <button onClick={() => handleAdd({
          id: 2, name: 'Mug', price: 15
        })}>
          Add Mug ($15)
        </button>
      </div>
      {optimisticCart.length === 0 ? (
        <p>Your cart is empty</p>
      ) : (
        <ul>
          {optimisticCart.map(item => (
            <li key={item.id}>
              {item.name} - ${item.price} ×
              {item.quantity}
              {' '}= ${item.price * item.quantity}
              <button
                onClick={() => handleRemove(item.id)}
                style={{ marginLeft: 8 }}
              >
                Remove
              </button>
              {item.pending && ' ...'}
            </li>
          ))}
        </ul>
      )}
      <p><strong>Total: ${total}</strong></p>
    </div>
  );
}
```

### Оптимистичное удаление с восстановлением после ошибки {/*optimistic-delete-with-error-recovery*/}

При оптимистичном удалении элементов нужно обрабатывать случай, когда Action завершается с ошибкой.

В этом примере показано, как вывести сообщение об ошибке, если удаление не удалось, а UI автоматически откатывается и снова показывает элемент.

```js tab="Tab 1"
//App.js
import { useState, startTransition } from 'react';
import { deleteItem } from './actions.js';
import ItemList from './ItemList';

export default function App() {
  const [items, setItems] = useState([
    { id: 1, name: 'Learn React' },
    { id: 2, name: 'Build an app' },
    { id: 3, name: 'Deploy to production' },
  ]);

  async function deleteAction(id) {
    await deleteItem(id);
    startTransition(() => {
      setItems(current => current.filter(item => item.id !== id));
    });
  }
```

```js tab="Tab 2"
//ItemList.js active
import { useState, useOptimistic, startTransition } from 'react';

export default function ItemList({ items, deleteAction }) {
  const [error, setError] = useState(null);
  const [optimisticItems, removeItem] = useOptimistic(
    items,
    (currentItems, idToRemove) =>
      currentItems.map(item =>
        item.id === idToRemove
          ? { ...item, deleting: true }
          : item
      )
  );

  function handleDelete(id) {
    setError(null);
    startTransition(async () => {
      removeItem(id);
      try {
        await deleteAction(id);
      } catch (e) {
        setError(e.message);
      }
    });
  }

  return (
    <div>
      <h2>Your Items</h2>
      <ul>
        {optimisticItems.map(item => (
          <li
            key={item.id}
            style={{
              opacity: item.deleting ? 0.5 : 1,
              textDecoration: item.deleting ? 'line-through' : 'none',
              transition: 'opacity 0.2s'
            }}
          >
            {item.name}
            <button
              onClick={() => handleDelete(item.id)}
              disabled={item.deleting}
              style={{ marginLeft: 8 }}
            >
              {item.deleting ? 'Deleting...' : 'Delete'}
            </button>
          </li>
        ))}
      </ul>
      {error && (
        <p style={{ color: 'red', padding: 8, background: '#fee' }}>
          {error}
        </p>
      )}
    </div>
  );
}
```

## Отладка {/*troubleshooting*/}

### Получаю ошибку: «An optimistic state update occurred outside a Transition or Action» {/*an-optimistic-state-update-occurred-outside-a-transition-or-action*/}

Вы можете увидеть такую ошибку:
```js
// 🚩 Incorrect: outside a Transition
function handleClick() {
  setOptimistic(newValue);  // Warning!
  // ...
}

// ✅ Correct: inside a Transition
function handleClick() {
  startTransition(async () => {
    setOptimistic(newValue);
    // ...
  });
}

// ✅ Also correct: inside an Action prop
function submitAction(formData) {
  setOptimistic(newValue);
  // ...
}
```

Когда вы вызываете setter вне Action, оптимистичное состояние ненадолго появляется, а затем сразу откатывается к исходному значению. Это происходит потому, что нет Transition, который мог бы «удерживать» оптимистичное состояние, пока выполняется Action.

### Получаю ошибку: «Cannot update optimistic state while rendering» {/*cannot-update-optimistic-state-while-rendering*/}

Вы можете увидеть такую ошибку:
```js
// 🚩 Incorrect: calling during render
function MyComponent({ items }) {
  const [isPending, setPending] = useOptimistic(false);

  // This runs during render - not allowed!
  setPending(true);

  // ...
}

// ✅ Correct: calling inside startTransition
function MyComponent({ items }) {
  const [isPending, setPending] = useOptimistic(false);

  function handleClick() {
    startTransition(() => {
      setPending(true);
      // ...
    });
  }

  // ...
}

// ✅ Also correct: calling from an Action
function MyComponent({ items }) {
  const [isPending, setPending] = useOptimistic(false);

  function action() {
    setPending(true);
    // ...
  }

  // ...
}
```

### Мои оптимистичные обновления показывают устаревшие значения {/*my-optimistic-updates-show-stale-values*/}

Если кажется, что оптимистичное состояние строится на старых данных, подумайте об updater-функции или reducer-е, чтобы вычислять оптимистичное состояние относительно текущего состояния.
```js
// May show stale data if state changes during Action
const [optimistic, setOptimistic] = useOptimistic(count);
setOptimistic(5);  // Always sets to 5, even if count changed

// Better: relative updates handle state changes correctly
const [optimistic, adjust] = useOptimistic(count, (current, delta) => current + delta);
adjust(1);  // Always adds 1 to whatever the current count is
```

См. [Обновление state на основе текущего состояния](#updating-state-based-on-current-state) для подробностей.

### Не знаю, находится ли моё оптимистичное обновление в pending {/*i-dont-know-if-my-optimistic-update-is-pending*/}

Чтобы понять, pending ли `useOptimistic`, у вас есть три варианта:
```js
const [optimistic, setOptimistic] = useOptimistic(value);
const isPending = optimistic !== value;
```

Если значения не равны, значит, Transition идёт.

2. **Добавить `useTransition`**
```js
const [isPending, startTransition] = useTransition();
const [optimistic, setOptimistic] = useOptimistic(value);

//...
startTransition(() => {
  setOptimistic(state);
})
```

Поскольку `useTransition` использует `useOptimistic` для `isPending` под капотом, это эквивалентно варианту 1.

3. **Добавить флаг `pending` в reducer**
```js
const [optimistic, addOptimistic] = useOptimistic(
  items,
  (state, newItem) => [...state, { ...newItem, isPending: true }]
);
```

Поскольку у каждого оптимистичного элемента есть собственный флаг, можно показывать состояние загрузки для отдельных элементов.
